# Glossary

## One-liner
Canonical terms used across the blog project.

## Terms

| Term | Definition |
|------|-----------|
| **Article** | A markdown document in a language-specific content folder, the atomic unit of the blog. |
| **Post** | Synonym for article (Hugo convention: `content/<lang>/posts/`). |
| **Series** | An ordered group of articles on a related topic, linked via `series` front matter. |
| **Translation** | A language variant of an article, generated by LLM and reviewed by the author. |
| **Primary language** | English — the language articles are authored in. |
| **Target languages** | French, Spanish, Mandarin Chinese, German, Japanese, Portuguese. |
| **Front matter** | YAML metadata block at the top of each markdown file. |
| **Hugo** | Static site generator used to build the blog. |
| **PaperMod** | Hugo theme providing the blog's visual design. |
| **giscus** | Comment widget backed by GitHub Discussions. |
| **Draft** | An article with `draft: true` — excluded from production builds. |
| **KB** | This knowledge base — the source of truth for the project. |
| **Ralph Loop** | Implement → validate → fix → repeat cycle. |

## Agent notes
> This glossary is the canonical source for naming. Use these exact terms in code, configs, and docs.
