我一直在思考，在 AI 智能体时代，软件工程正在发生怎样的变化。

我曾在 X 上和一些演讲中分享过其中的一些想法，但我想要一个地方来更充分地展开它们——写更长的文章、引用他人的观点、获取反馈，并随着时间的推移积累一系列作品。

这个博客就是那个地方。

## 期待什么

我会写关于：

- **智能体驱动的开发** —— 如何让 AI 智能体构建正确的软件，而不仅仅是看起来合理的软件。
- **规约驱动的方法论** —— 为什么当你的开发者是随机的时候，规约反而变得更重要了。
- **知识工程** —— 构建智能体能够真正驾驭的上下文。
- **编程的未来** —— 什么在改变，什么保持不变，以及我们应该关注什么。

## 这个博客如何运作

这个博客使用 Hugo 构建，部署在 GitHub Pages 上。[源代码是公开的](https://github.com/yurug/blog)——你可以在 git 历史中看到每一次更改。

文章提供多种语言版本，借助 LLM 翻译，并由我审核。

你可以通过评论区对任何文章留下反馈，评论区由 GitHub Discussions 驱动。

欢迎加入。
